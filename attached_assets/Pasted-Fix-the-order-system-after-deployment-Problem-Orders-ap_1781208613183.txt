Fix the order system after deployment.

Problem:
Orders appear in admin before publishing, but after publishing on Replit, customer orders do not appear in admin dashboard.

Likely cause:
Orders are stored in localStorage/client-side storage. This only works in the same browser/device and does not share data between customers and admin.

Required fix:
Move all order storage to a real shared backend/database.

Do not rebuild the website from scratch.
Inspect the current code first and find where orders are created, saved, read, and displayed in admin.

Use Supabase as the shared database.

Tasks:

2. Create .env.example with:

4. Checkout flow:
When customer places order:
- Require login.
- Validate address fields.
- Save order to Supabase orders table.
- Save products to order_items table.
- Do NOT use WhatsApp.
- Do NOT save only in localStorage.
- Show success message:
“Order placed successfully. We’ll contact you to confirm.”

Show all customer orders from all devices.

Out for Delivery

6. When admin updates order status:

- Customer can see status update in My Orders/Profile.
